## Заголовок
Невозможно перегенерировать поля с функцией повтороной генерации в Рекомендации_Статегии
## Предусловия 
- Осуществлен вход со своего личного аккаунта на сайте https://example.com
- Создан блог в вкладке "Сайт"(иконка стека), для блога пройден опрос и на его основе сгенерированна стратегия
- Пользователь находится во вкладке Сайт -> Редактировать(рядом с названием блога) -> **Стратегия(иконка планшета)**
## Шаги
1. Нажать на панель "Мой бренд" (Рекомендации/стратегия)
2. Справа от одного из следующих заголовков: "Целевая аудитория", "Сегменты ЦА", "Позиционирование", "Tone of Voice"; нажать на кнопку перегенерации (завернутая стрелочка).
## Фактический результат
Содержимое поля/полей под заголовком не изменилось. В левом нижнем углу всплыл попап вида "Задача с id 6686b251-12b9-45b0-bff6-56ffffdd966d провалилась".
Данные ответа:
```HTTP
Request URL
https://example/api/v1/org/linuxblog/strategy/regenerate

Request Method
POST

Status Code
202 Accepted

Remote Address
92.00.000.00:443

Referrer Policy
strict-origin-when-cross-origin
```
Тело ответа:
```json
{"task_id":"6686b251-12b9-45b0-bff6-56ffffdd966d","status":"pending"}
```
## Ожидаемый результат
Содержимое поля генерируется заново, статус код ответа 200 ОК. Тело ответа содержит json strategy с перегенерированными полями. Попапов сообщающих об ошибке не появляется.
## Окружение
Устройство и система: PC, Windows
Браузер: Chrome
