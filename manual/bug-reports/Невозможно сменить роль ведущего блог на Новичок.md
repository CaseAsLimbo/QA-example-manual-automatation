## Заголовок
Невозможно сменить роль ведущего блог на Новичок.
## Предусловия
- Осуществлен вход со своего личного аккаунта на сайте https://example/editor/
- Создан блог в вкладке "Сайт"(иконка стека), для блога пройден опрос и на его основе сгенерированна стратегия
- Для блога определена любая роль кроме Новичок
- Пользователь находится во вкладке Сайт -> Редактировать(рядом с названием блога) -> **Стратегия(иконка планшета)**
## Шаги
1. Нажать на панель "Мой бренд" (Рекомендации/стратегия)
2. Справа от заголовка "Роль в блоге" нажать на кнопку редактирования (иконка карандаша)
3. В появившемся окне несколько раз нажать на кнопку выбора следующей роли вида "Роль >", до тех пор пока не будет показана роль Новичок и нажать на кнопку "Выбрать"

## Фактический результат
Роль переключается на Новичок. В левом нижнем углу всплывает попап "Не удалось сохранить изменения". Статус код 422, метода PATCH. После перезагрузки страницы роль меняется на выбранную ранее.
Данные ответа:
```HTTP
Request URL
https://example/api/v1/org/linuxblog/strategy

Request Method
PATCH

Status Code
422 Unprocessable Content

Remote Address
90.00.000.00:443

Referrer Policy
strict-origin-when-cross-origin
```
Тело ответа:
```JSON
{
    "detail": [
        {
            "type": "enum",
            "loc": [
                "body",
                "role"
            ],
            "msg": "Input should be 'professor', 'star', 'creator', 'preacher' or 'newbie'",
            "input": "beginner",
            "ctx": {
                "expected": "'professor', 'star', 'creator', 'preacher' or 'newbie'"
            }
        }
    ]
}
```
